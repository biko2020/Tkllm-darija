Database schema — Prisma setup
  • Define schema.prisma with core models: User, Contributor, Task, DataRecord, QualityReview, Payment.
  • Enable timescaledb extension via prisma/migrations — add a TimescaleDB hypertable for DataRecord with a time column.
  • Configure PostGIS for regional/accent geo-metadata (contributor_location column).
  • Run prisma migrate dev against the local Docker Postgres instance; verify schema.
  • Add a seed.ts with representative fixture data for local development.
API skeleton — NestJS modules
  • Create core modules: UserModule, AuthModule, TaskModule, DataModule, QualityModule, PaymentModule.
  • Wire a global ConfigModule using @nestjs/config + Joi schema validation for all env vars.
  • Set up PrismaModule as a global provider with connection pooling and health checks.
  • Implement RedisModule using ioredis with a shared connection provider.
  • Add global exception filter (HttpExceptionFilter), logging interceptor, and response transformation interceptor.
  • Expose /health endpoint using @nestjs/terminus — check DB, Redis, and disk.
GraphQL layer
  • Configure GraphQLModule with code-first approach, auto-schema generation, and playground disabled in production.
  • Define User and Task object types; stub resolvers for all CRUD operations.
  • Add DataLoader pattern (via nestjs-dataloader) to prevent N+1 query issues.
Docker local stack
  • Complete infrastructure/docker/docker-compose.yml — add PostgreSQL+TimescaleDB, Redis, MinIO, Kafka+Zookeeper, MailHog, pgAdmin.
  • Write infrastructure/docker/init-scripts/postgres/01_extensions.sql to enable all required extensions on startup.
  • Add infrastructure/docker/minio/buckets.json to auto-create tkllm-audio, tkllm-datasets, tkllm-models buckets.
  • Verify infrastructure/scripts/setup-local.sh runs the full stack in one command.

✓
M1 — API starts locally, /health returns green for all services, Prisma migrations run cleanly, GraphQL playground accessible.
02 Authentication & User Management
Wk 7–9
Auth stack

@nestjs/passportStrategy management

Authentication flows
  • Implement POST /auth/register with phone number (Moroccan +212 format) or email, bcrypt hashed password, and email/SMS verification flow.
  • Implement POST /auth/login returning short-lived access token (15 min) + long-lived refresh token (30 days) stored in Redis.
  • Implement POST /auth/refresh with token rotation — old refresh token invalidated on use.
  • Implement POST /auth/logout — blacklist token in Redis with TTL.
  • Add JwtAuthGuard and RolesGuard decorators; define roles: CONTRIBUTOR, ADMIN, B2B_CLIENT.
  • Implement OTP-based phone verification: generate 6-digit code, store in Redis with 10-min TTL, send via SMS.
Consent & CNDP compliance (Morocco Law 09-08)
  • Model ConsentRecord in Prisma with category, version, granted_at, revoked_at fields.
  • Implement POST /users/consent — record granular consent per data category (audio, text, metadata, location).
  • Implement DELETE /users/consent/:category — revoke specific category; cascade data anonymization job to BullMQ queue.
  • Add audit log interceptor that records all data access events to a separate audit table.
Contributor profile management
  • Implement CRUD endpoints for ContributorProfile: region, age group, gender, primary dialect, native language.
  • Add profile completeness scoring — contributors with complete profiles get priority task assignment.
  • Implement GET /users/me with aggregated stats (tasks completed, earnings, quality score).
Milestone
✓
M2 — Full auth flow tested end-to-end, role-based access working, consent records stored with audit trail, phone OTP functional.
03 Data Collection, Task Engine & Quality
Wk 10–14
Task engine — core logic
  • Model TaskTemplate (domain, type, prompt content, language, difficulty) and TaskAssignment (contributor, status, assigned_at, submitted_at) in Prisma.
  • Implement contributor matching algorithm: filter by dialect, region, completed task history, and current workload to distribute tasks fairly.
  • Implement GET /tasks/next — return the best matching unassigned task for the authenticated contributor.
  • Implement POST /tasks/:id/submit — accept audio, text, or annotation payload; validate schema; enqueue for quality review.
  • Implement task expiration: BullMQ job that re-assigns tasks after 24h if not submitted.
Audio upload pipeline
  • Add @nestjs/platform-fastify with fastify-multipart for efficient file upload handling.
  • Validate audio: enforce WAV/MP3/OGG, max 60s duration, min 8kHz sample rate using music-metadata.
  • Upload to MinIO/S3 with server-side AES-256 encryption; store only the object key in the DB.
  • Generate a pre-signed URL (5-min TTL) for playback — never expose raw bucket URLs.
  • Enqueue ASR_TRANSCRIBE job in BullMQ after successful upload.
Quality engine module
  • Implement multi-stage review: automated checks (audio SNR, duration, encoding) → AI pre-screening → human review queue.
  • Model quality scores (0–100) with breakdown: audio clarity, transcription accuracy, annotation completeness.
  • Implement majority voting for text annotation tasks — require ≥ 3 independent annotations; flag disagreements for senior review.
  • Implement contributor reputation scoring: rolling 30-day average quality score; auto-suspend accounts below threshold 40.
  • Create GET /admin/quality/review-queue for admin review of flagged submissions.
Data schemas — implement & validate
  • Write data/schemas/audio_annotation.json, quality_review.json, dataset_metadata.json as JSON Schema drafts.
  • Mirror schemas as Zod objects in packages/validators for runtime validation in the API.
  • Write data ingestion scripts: data/ingestion/doda_ingest.py, dvoice_ingest.py, atlasia_ingest.py — download, normalize, and load into PostgreSQL.
Milestone
✓
M3a — Task assignment and submission flow working end-to-end with audio upload to MinIO.
✓
M3b — Quality scoring pipeline running, contributor reputation scores updating after each review.
04 ML Integration, Async Services & Payments
Wk 15–18
Kafka event bus setup
  • Complete infrastructure/messaging/kafka/topics.yml: define topics audio.uploaded, task.submitted, quality.reviewed, payment.triggered, dataset.published.
  • Register @nestjs/microservices Kafka client in the API; emit domain events on each state transition.
  • Add dead-letter topics (*.dlq) for all production topics; configure retry policy (3 attempts, exponential backoff).
  • Write infrastructure/scripts/seed-kafka-topics.sh to create topics in local and CI environments.
ASR worker service (services/asr-worker)
  • Set up Python FastAPI service that consumes audio.uploaded Kafka topic.
  • Load Whisper large-v3 model at startup; implement transcribe(audio_url) function with pre-signed S3 download.
  • Return transcription result + confidence score; publish to task.transcribed topic.
  • Containerize with GPU-capable Dockerfile (CUDA 12.1 base image); add KEDA ScaledObject to scale on Kafka lag.
Data pipeline service (services/data-pipeline)
  • Implement Prefect flows for: dataset export (Parquet / JSONL), dataset versioning via DVC, and dataset registry update.
  • Add data/registry/datasets.yaml and auto-update script data/ingestion/update_summary.py.
  • Implement POST /admin/datasets/export API endpoint that triggers a Prefect flow run.
Financial service (services/financial-service)
  • Implement contributor wallet model: credits accrued per approved task, real-money conversion on withdrawal request.
  • Implement Orange Money and Inwi Money adapter stubs with a provider interface — use sandbox credentials in dev.
  • Implement payout logic: batch payouts nightly via BullMQ scheduled job; enforce minimum threshold (50 MAD).
  • Implement basic fraud detection rules: velocity checks (>50 submissions/hour), duplicate audio hash detection, IP anomaly flagging.
  • All financial operations must be idempotent — use idempotency_key column in Transaction table.
Analytics service
  • Implement TimescaleDB continuous aggregates for: daily active contributors, tasks submitted per hour, quality score trends.
  • Expose GET /admin/analytics/dashboard endpoint returning pre-aggregated metrics.
  • Wire Prometheus metrics exporter in the NestJS API (@willsoto/nestjs-prometheus).
Gamification & rewards
  • Implement streak tracking: Redis sorted sets for daily activity; reset on missed day.
  • Implement leaderboard: Redis sorted set keyed by weekly earnings; GET /leaderboard/weekly returns top 100.
  • Implement achievement badges: stored in JSONB column, awarded via event listeners on task milestones.

✓
M4a — End-to-end flow: audio uploaded → Kafka event → ASR worker transcribes → result stored → quality reviewed → payment triggered.
✓
M4b — Payout sandbox working, fraud rules active, leaderboard and streaks updating in real time.
05 Testing & Validation
Wk 17–20 (parallel)

Unit testing — targets per module
  • AuthService — token generation, refresh rotation, OTP expiry edge cases, role guard behavior.
  • TaskEngine — contributor matching algorithm, expiry logic, duplicate submission rejection.
  • QualityEngine — scoring algorithm correctness, majority vote calculation, reputation threshold enforcement.
  • FinancialService — wallet arithmetic, idempotency key deduplication, fraud rule triggers.
  • Achieve minimum 80% line coverage; enforce via CI coverage gate.
Integration & E2E testing
  • Use Testcontainers to spin up real PostgreSQL and Redis instances in CI — no mocks for DB layer.
  • Write E2E test suite covering the critical path: register → verify phone → get task → submit audio → receive quality score → trigger payout.
  • Test all error paths: invalid file formats, expired tokens, over-quota submissions, duplicate idempotency keys.
  • Write Kafka consumer integration tests using an embedded Kafka broker.
Load testing with k6
  • Define SLO targets: POST /tasks/:id/submit p95 < 500ms at 200 rps; GET /tasks/next p99 < 200ms.
  • Run ramp test: 0 → 500 virtual users over 5 min; record throughput, error rate, and DB connection pool saturation.
  • Identify and fix top 3 slow queries using EXPLAIN ANALYZE; add missing indexes.
Security validation
  • Run OWASP ZAP baseline scan against the staging API; remediate all HIGH findings before prod deployment.
  • Manually verify: JWT secret rotation, rate limiting on auth endpoints (max 10 req/min), SQL injection protection via Prisma parameterization, CORS policy.
  • Verify GDPR/CNDP data deletion: trigger consent revocation and confirm audio files deleted from S3, PII pseudonymized in DB within 30 seconds.
ML evaluation
  • Complete ml/evaluation/evaluate_asr.py — run WER and CER on a held-out Darija test set.
  • Complete ml/experiments/mlflow.yaml and log all experiment runs to MLflow server.
  • Document baseline model performance in docs/ml-baseline.md.
Milestone
✓
M5 — 80%+ test coverage, all E2E tests green, p95 latency within SLO, zero OWASP HIGH findings.
⚠ Testing runs in parallel with phases 3 & 4 — write tests alongside each module, not after. Retrofitting tests onto complex async code is significantly harder.
06 CI/CD & Deployment
Wk 19–22
CI/CD pipelines — GitHub Actions
  • Add .github/workflows/api.yml: lint → test → build Docker image → push to ECR → deploy to staging.
  • Add .github/workflows/infra.yml: terraform fmt check → terraform validate → plan on PR → apply on merge to main (staging only; prod requires manual approval).
  • Add .github/workflows/ml.yml (already started): lint Python → run pytest → evaluate model if training data changed.
  • Enforce branch protection: require 1 review + all status checks green before merge to main.
  • Add Dependabot for automated dependency updates (weekly, grouped by ecosystem).
Docker & container registry
  • Write production Dockerfile for apps/api: multi-stage build (build stage → slim runtime), non-root user, HEALTHCHECK directive.
  • Write Dockerfiles for each service in services/ — ASR worker uses CUDA base image.
  • Complete Terraform infrastructure/terraform/modules/ecr/ — create repositories for all service images.
  • Tag images with Git SHA; never push :latest to production ECR.
Terraform — provision staging environment
  • Complete infrastructure/terraform/environments/staging/terraform.tfvars — medium-sized resources, single-AZ, no multi-AZ RDS.
  • Run terraform apply for staging: VPC, EKS cluster, RDS PostgreSQL, ElastiCache Redis, MSK Kafka, S3 buckets, SSM parameters.
  • Verify all outputs: cluster endpoint, DB connection string, bucket names — store in GitHub Secrets.
Kubernetes — staging deployment
  • Complete infrastructure/k8s/base/: namespace, RBAC, network policies, ConfigMaps, ExternalSecrets CRDs.
  • Complete deployment manifests for API, ASR worker, quality-engine, data-pipeline, financial-service.
  • Apply infrastructure/k8s/overlays/staging/ — lower replica counts, dev image tags, relaxed resource limits.
  • Verify all pods reach Running; confirm inter-service communication via ClusterIP services.
  • Configure NGINX ingress with TLS via cert-manager + Let's Encrypt for staging domain.
Monitoring stack
  • Deploy Prometheus + Grafana via Helm; configure infrastructure/monitoring/prometheus/prometheus.yml scrape targets.
  • Create the 4 Grafana dashboards: api-overview.json, infrastructure.json, ml-pipeline.json, business.json.
  • Configure alerting rules in infrastructure/monitoring/prometheus/rules/ for API error rate, DB lag, payment failures.
  • Integrate Sentry SDK in the NestJS API for runtime error tracking.
Milestone
✓
M6a — CI pipeline runs on every PR; Docker images pushed to ECR automatically.
✓
M6b — Full stack running on staging K8s; Grafana dashboards showing live data; Sentry receiving errors.
07 Production Hardening & Final Delivery
Wk 23–26
Production infrastructure
  • Complete infrastructure/terraform/environments/prod/terraform.tfvars — multi-AZ RDS, larger EKS node groups with GPU nodes, stricter security groups.
  • Apply production Terraform — require manual approval in GitHub Actions workflow (environment: production with required reviewer).
  • Enable RDS automated backups (7-day retention), point-in-time recovery, and cross-region snapshot replication.
  • Configure Cloudflare DNS and WAF rules via Terraform cloudflare module — enable DDoS protection and bot management.
Security hardening
  • Apply infrastructure/k8s/overlays/prod/network-policies/ — restrict all egress to explicit allow-list.
  • Enable Pod Security Admission (restricted profile) for all non-system namespaces.
  • Rotate all secrets via infrastructure/scripts/rotate-secrets.sh; store new values in AWS Secrets Manager, pull via ExternalSecrets operator.
  • Enable AWS GuardDuty and CloudTrail on the production account for threat detection and audit logging.
  • Run a full penetration test using OWASP ZAP authenticated scan; document and remediate all findings.
Documentation — complete all missing files
  • Write docs/local-setup.md — step-by-step guide covering all prerequisites, Docker stack, seed data, and common troubleshooting.
  • Write CONTRIBUTING.md — branch naming, commit conventions, PR template, coding standards, test requirements.
  • Generate OpenAPI spec from NestJS decorators (@nestjs/swagger); publish at /api-docs in staging only.
  • Write docs/ml-baseline.md — model performance metrics, dataset sources, and fine-tuning instructions.
  • Complete data/registry/lineage/v1.yaml — document provenance of all ingested datasets.
Production go-live checklist
  • Run infrastructure/scripts/health-check.sh against production — all services must return healthy.
  • Execute smoke tests on production: register → submit audio → confirm quality score appears → confirm wallet updated.
  • Verify all Grafana SLO dashboards are green; set up PagerDuty or equivalent on-call rotation for P1 alerts.
  • Perform a dry-run of the disaster recovery procedure: simulate RDS failover, verify API recovers within defined RTO.
  • Tag the release: git tag v1.0.0; create GitHub Release with changelog generated from conventional commits.
Post-launch — first 30 days
  • Monitor contributor onboarding funnel daily — track drop-off at phone verification, first task, and first payout.
  • Set up weekly ML model evaluation cron: compute WER on new gold-standard transcriptions added by the quality team.
  • Hold weekly retrospectives; triage any Sentry errors that exceed 10 occurrences/day.
  • Initiate data/versions/v2/ dataset snapshot with first month of platform data.
Milestone
✓
M7 — Production live — All services healthy, documentation complete, SLOs met, on-call rotation active, v1.0.0 tagged.
✓ Project delivered. The Tkllm-Darija API is in production, monitored, documented, and ready to onboard contributors and B2B customers.
